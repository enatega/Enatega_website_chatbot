# Getting Started

## Introduction
Enatega Multivendor is a full-featured multivendor food delivery solution for iOS, Android, and the Web. We have ensured that you receive a good mobile and dashboard application design as well as a complete solution that will allow you to quickly integrate and white label into any meal delivery service.

## Technology Overview
The mobile end of this application is built utilizing React Native and Expo. React has been used for the dashboard panel and customer web app. GraphQL is utilized by both web and mobile, with the Apollo Client used for endpoint querying and state management. Node with MongoDB is used to create the API. 

### Key Features
The application includes all the features necessary for any restaurant or food delivery application. Some of the features are:

#### Customer Mobile/Web Application
- Verification of email and phone numbers
- Authentication using Google, Apple, and Facebook
- Restaurants displayed on the map and home screen based on location
- Information about restaurants, including reviews, ratings, hours of operation, delivery schedules, menu items, and establishment location
- Address addition using Maps integration and Google Places suggestions
- Analytics and reporting on errors with Amplitude and Sentry
- Push notifications and emails for account creation and order status updates, including global push notifications
- Real-time tracking of riders and chat options with riders
- Multi-language and theme support
- Rating and review features for orders
- Payment integration for both PayPal and Stripe
- Previous order history and favorite restaurant options
- Variations of food items and notes to restaurants
- Pick-up and delivery options with different timings
- Vouchers and tipping options
- Search functionality for restaurants and their items

#### Restaurant Application
- Ringer and push notification alerts for new incoming orders
- Time limiter for order acceptance and meal preparation settings
- Print invoice option for restaurants
- Delivered orders history
- Online/offline status management

#### Rider Application
- Real-time order status change updates
- Push notification and ringer feature for new orders
- Map feature showing delivery and restaurant addresses with Google Maps integration
- Time limiter for order acceptance and updated meal preparation time
- Display of distance and anticipated travel time
- Real-time chat with customers, including push notifications for chats
- Order history and new orders
- Online delivery fees management and wallet history
- Online/offline status management for riders

#### Admin Dashboard
- Role-based administration, granting the administrator access to all features while vendors manage their own establishments
- Management of restaurants, vendors, and restaurant sections
- Defining zones for riders and assigning them
- Managing withdrawal requests from riders and commission rates
- Configuration of application and global order status management
- Restaurant order statistics and management of Stripe accounts for commission
- Menu management with discount features
- User, ratings, coupon, and tipping option management

## Functionalities for Better Development Experience
To enhance the development experience, the code is neatly organized, and unnecessary screens have been removed. The following functionalities are included:
- ESLint for JavaScript linting
- Prettier for code formatting
- Jest for unit testing
- Husky to prevent bad commits

## Enatega Technology Stack
- **Application Program Interface (API) Server**: NodeJS, MongoDB, ExpressJS, Stripe, PayPal, Nodemailer, Firebase (for web push notifications), Express, GraphQL, Mongoose (for MongoDB)
- **Web Dashboard**: React, GraphQL, Bootstrap, Firebase (for web push notifications)
- **Mobile App**: React Native, Expo, GraphQL, Sentry, Amplitude
- **Rider App**: React Native, Expo, GraphQL, Sentry
- **Restaurant App**: React Native, Expo, GraphQL, Sentry
- **Customer Web App**: React, Material UI, GraphQL, Amplitude, Sentry, Firebase (for web push notifications)

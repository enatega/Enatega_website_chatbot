## Docker Setup
### Step 1: Check Docker Installation
Ensure Docker is installed on your machine. If not, download it from the official website.

### Navigate to Project Directory
After installing Docker, open your terminal and navigate to the project directory.

### Step 2: Environment Variables
After setting up Docker, configure the environment variables by modifying or creating the `.env` file with your credentials. If the `.env` file does not exist, create one in the root directory and include the following contents:

```
PORT=8001
CONNECTION_STRING=
RESET_PASSWORD_LINK=auth/reset/?reset=
SERVER_URL=https://enatega-multivendor.up.railway.app/
STRIPE_WEBHOOK_ENDPOINT_SECRET=
DASHBOARD_URL=https://practical-minsky-95a652.netlify.app/#/
WEB_URL=https://pensive-liskov-5320d4.netlify.app/
ORDER_DETAIL_WEB_URL=order-detail/
NODE_DEV=production
NODE_ENV=production
SENDGRID_API_KEY=
```

### Environment Variables Explanation
- **PORT**: The port number on which the server will run.
- **CONNECTION_STRING**: Contains the connection string for your MongoDB database.
- **RESET_PASSWORD_LINK**: Link for resetting passwords.
- **SERVER_URL**: Base URL of your server.
- **STRIPE_WEBHOOK_ENDPOINT_SECRET**: Used for Stripe webhook verification.
- **DASHBOARD_URL**: URL for the dashboard.
- **WEB_URL**: URL for the web application.
- **ORDER_DETAIL_WEB_URL**: Partial URL for specific order details.
- **NODE_DEV**: Node development environment.
- **NODE_ENV**: Node environment.
- **SENDGRID_API_KEY**: Enter your SendGrid API key if using SendGrid for emails.

### Step 3: Adding ServiceAccountKey.json
To add your `serviceAccountKey.json`:
1. Go to the Firebase Console.
2. Select your project.
3. Navigate to Project Settings > Service accounts.
4. Click on “Generate new private key” to download the JSON file.
5. Replace the downloaded `serviceAccountKey.json` file in the project directory.

## Running The Backend
After completing the setup steps, you can start using your backend API. Simply run Docker, and your API will be ready to serve your application. To build and start the Docker containers, run the following command:

```
docker compose up --build
```
